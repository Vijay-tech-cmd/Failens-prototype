# FairLens — AI Bias Audit Tool

> Detect unfairness in machine learning models in under 60 seconds.

FairLens is an AI-powered bias auditing platform built for the **Google Solution Challenge 2025** under the **Unbiased AI Decision** problem statement. Upload any tabular dataset and trained ML model — FairLens computes fairness metrics across demographic groups, visualizes the gaps, and generates a plain-English audit report powered by Google Gemini.

---

## The Problem

AI systems are making life-changing decisions — who gets hired, who gets a loan, who gets approved. These models are trained on historical data that reflects real-world discrimination. The result: biased AI that nobody audits.

- Amazon's hiring AI penalized women's resumes for **3 years** before anyone noticed
- COMPAS (used in US courts) flagged Black defendants as high-risk at **2× the rate** of white defendants
- Loan approval AI shows systematic gaps across racial groups with no transparency

There was no simple tool to catch this. Until now.

---

## What FairLens Does

1. **Upload** your CSV dataset and trained model (.pkl)
2. **Select** which demographic column to audit (gender, race, age)
3. **Analyze** — FairLens runs 3 fairness metrics in seconds
4. **Read** the AI-generated plain-English report with fix recommendations

## Fairness Metrics Computed

| Metric | What it measures | Threshold |
|---|---|---|
| Demographic Parity | Are approval rates equal across groups? | Gap > 5% = FAIL |
| Equalized Odds | Are true positive and false positive rates equal? | Gap > 5% = FAIL |
| Per-group Accuracy | Does the model perform equally well for all groups? | Displayed for reference |

## Local Setup

### Prerequisites
- Python 3.10+
- Node.js 18+
- A Gemini API key from [aistudio.google.com](https://aistudio.google.com)

### Backend

```bash
# 1. Clone the repo
git clone https://github.com/YOUR_USERNAME/fairlens.git
cd fairlens/backend

# 2. Create and activate virtual environment
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux

# 3. Install dependencies
pip install -r requirements.txt

# 4. Create your .env file
echo GEMINI_API_KEY=your_key_here > .env

# 5. Download the demo dataset (run once)
python get_data.py

# 6. Train the demo model (run once)
python train_model.py

# 7. Start the API server
uvicorn main:app --reload
```

Backend runs at: `http://localhost:8000`
Swagger docs at: `http://localhost:8000/docs`

### Frontend

```bash
# In a new terminal
cd fairlens/frontend

# Install dependencies
npm install

# Start dev server
npm run dev
```

Frontend runs at: `http://localhost:5173`

## API Reference

### `POST /audit`

Runs a full bias audit on the uploaded model and dataset.

**Request** — `multipart/form-data`

| Field | Type | Description |
|---|---|---|
| `dataset` | file (.csv) | The dataset used to evaluate the model |
| `model_file` | file (.pkl) | Trained scikit-learn model |
| `sensitive_col` | string | Column to audit (e.g. `sex`, `race`) |
| `label_col` | string | Target column (e.g. `income`) |

**Response**

```json
{
  "demographic_parity": {
    "gap_percent": 18.4,
    "status": "FAIL",
    "group_rates": {
      "Female": 11.2,
      "Male": 29.6
    }
  },
  "equalized_odds": {
    "gap_percent": 12.3,
    "status": "FAIL",
    "tpr_by_group": { "Female": 55.1, "Male": 67.4 },
    "fpr_by_group": { "Female": 4.2,  "Male": 16.5 }
  },
  "group_accuracy": {
    "Female": 88.1,
    "Male": 84.3
  },
  "ai_report": "The audit found a significant disparity...",
  "total_rows": 48842,
  "sensitive_column": "sex"
}
```
